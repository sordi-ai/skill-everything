Skill-Everythink

"Agents that never make the same mistake twice"

Git-Versioned Agent Memory — The intelligent fine-tuning replacement

Open Source • MIT License • Agent-agnostic

The Problem

AI agents forget everything. Every session starts from zero. Existing solutions have fundamental weaknesses:

  • 💸 Fine-Tuning — Expensive ($$$), rigid, outdates quickly, vendor lock-in
  • 🏗️ RAG — Requires infrastructure: vector databases, embedding APIs, maintenance
  • 📄 Custom Instructions — Static, unversioned, no structure
  • 🔒 Cursor Rules — Cursor-only, no memory, no learning curve from errors
  • 🧊 mem0 / MemGPT — Opaque (you can't see what's stored), API-dependent

→ None of these approaches LEARN from mistakes.

The Solution: Skill-Everythink

  • ✅ Living skill system from plain Markdown — no setup, no API key
  • ✅ Error memory — Every mistake is captured, analyzed, turned into a rule
  • ✅ Domain knowledge — Processes, rules, internal terms, partners, architecture
  • ✅ Git-versioned — Every change traceable, reviewable, rollback-capable
  • ✅ Agent-agnostic — OpenCode, Claude Code, Gemini CLI, Cursor, any agent that reads context
  • ✅ Modular — Sub-skills for different domains, grows with the team

→ Like fine-tuning, but free, flexible, and transparent.

Self-Extension

The agent extends the skill automatically:

1
Detect mistake — Test fails, review comment, wrong approach
2
Determine category — Development, Git, Deployment, Security, Domain?
3
"Search before Write" — Does a similar rule already exist? Update instead of duplicate.
4
Formulate rule — Action directive, not description
5
Insert into target file — At the end of the matching section
6
Update error log — Complete YAML entry with context
